# 官网 / 文档 / 白皮书资料域采集设计

## 1. 目标

当前采集层对官网和文档的处理仍偏单页抓取：给一个 URL，就抓这一页。  
这不足以支撑项目基本面分析，因为真实项目的重要信息通常分散在多个页面、多个章节或多个文档入口下。

本设计的目标是把采集单位从“单页面”升级为“资料域快照”：

- 官网：形成页面集合快照
- Docs：形成目录 + 页面集合快照
- 白皮书：形成章节快照

采集层不提前写死“哪些页面最重要”，而是尽量受控收集相关资料，再交由后续 LLM 判断哪些内容真正重要。

## 2. 设计原则

- 采集层负责“尽量收全”，不负责“最终判断”
- 页面价值不在采集层写死，避免误删项目特有的重要页面
- 通过页面数量上限控制范围，而不是通过固定页面名单控制
- 保留页面关系、发现路径、原始内容和清洗文本，形成完整资料域快照
- 官网、Docs、白皮书分别建模，不能一套逻辑通吃

## 3. 采集层分层

### L1. 入口识别

输入可以是：

- 官网 URL
- Docs URL
- 白皮书 PDF URL
- 上传的 PDF 文件
- 某个官网/Docs 二级页

系统需要先识别：

- 该入口属于 `website` / `docs` / `whitepaper_pdf`
- 它属于哪个资料域
- 它是否是入口页、目录页、单页文档或二级页

### L2. 资料域发现

系统围绕给定入口在资料域内扩展候选页面：

- 官网：同域内部链接
- Docs：同 docs 域链接、目录页、章节页
- 白皮书：目录、章节、页码结构

本层目标是“尽量发现”，不做强价值裁剪。

### L3. 资料域快照采集

系统对已发现的候选页面进行受控采集，形成资料域快照。

控制方式：

- 官网资料域：建议上限 `30` 页
- Docs / 白皮书资料域：建议上限 `50` 页
- 命中多语言镜像时优先主语言
- 去重后再进入下一阶段

## 4. 资料域快照对象

每个被采集到的页面或章节，至少保存以下信息：

### 4.1 页面基础信息

- `url`
- `title`
- `domain`
- `source_domain_type`
- `discovered_from`

### 4.2 页面内容信息

- `raw_html` 或原始文档内容
- `clean_text`
- `summary`
- `content_length`

### 4.3 页面结构信息

- 内部链接列表
- 被谁发现 / 被谁链接
- docs 父子目录关系
- PDF 页码 / 章节映射

### 4.4 页面状态信息

- 抓取状态
- 抓取时间
- 是否重复
- 是否被截断
- 页面语言

### 4.5 轻量标签

仅保留基础标签，不做最终价值判断：

- 是否首页
- 是否 docs index
- 是否 PDF
- 是否 tokenomics 风格页
- 是否 roadmap 风格页
- 是否 FAQ / support 页

## 5. 三类资料域的区别

### 5.1 官网资料域

特点：

- 页面数量通常较少
- 导航层级不深
- 价值集中在少数核心页
- 容易混入活动页、营销页、落地页

采集目标：

- 建立官网页面集合快照
- 保留导航与链接关系
- 不在采集阶段删掉“看起来不重要”的页面

### 5.2 Docs 资料域

特点：

- 页面数量可能很多
- 结构性强
- 常见目录页、子章节页、索引页
- 容易出现模板化重复页面

采集目标：

- 建立目录结构 + 页面集合快照
- 保留父子关系和目录层级
- 优先保持文档结构完整性

### 5.3 白皮书资料域

特点：

- 通常是单文档
- 重点不在链接扩展，而在章节化理解

采集目标：

- 完整抓全文
- 保留 metadata
- 保留页码、章节、段落映射
- 建立章节快照，而不是网页集合

## 6. 与后续判断层的边界

本设计只定义采集层，不定义最终分析判断。

采集层的职责：

- 收集资料域内容
- 保留页面/章节结构
- 形成资料域快照

判断层（后续 LLM）的职责：

- 判断哪些页面真正关键
- 去除重复或低价值内容
- 提炼关键机制、风险、机会点

## 7. 第一阶段落地建议

建议按以下顺序实施：

1. 官网资料域扩展采集
2. Docs 资料域扩展采集
3. 白皮书章节化采集
4. 资料域快照统一读模型
5. LLM 页面重要性判断

## 8. 结论

官网、Docs、白皮书不应再被视作“单页抓取对象”，而应升级为“资料域快照对象”。  
采集层的正确方向是：

- 受控全采集
- 不提前写死高价值页面
- 通过页面数量上限控制范围
- 把“哪些内容真正重要”交给后续判断层
